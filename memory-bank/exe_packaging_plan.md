# コエモジ - EXE化計画

## 目的
コエモジをPyInstallerを使用して実行可能ファイル（EXE）に変換し、Python環境がインストールされていないエンドユーザーでも簡単に利用できるようにする。

## EXE化の詳細設計

### 1. パッケージング戦略
- **パッケージング方式**: ディレクトリ形式
  - **利点**: 
    - 起動が速い（単一ファイル形式より）
    - FFmpegなどの外部バイナリを効率的に扱える
    - リソースファイルの参照が容易
  - **欠点**:
    - 配布時に複数ファイルの管理が必要
    - 単一ファイル形式より少し複雑

### 2. リソース管理計画
- **アプリケーションアイコン**:
  - 使用するアイコン: `resources/koemoji-logo256x256.ico`
  - PyInstallerのicon引数で指定

- **リソースファイル**:
  - `resources/`フォルダ内のすべての画像ファイル
  - 実行時のパス解決方法を適切に設定

- **FFmpeg**:
  - `ffmpeg_bin/`フォルダをそのまま同梱
  - 現在のパス解決ロジックを維持（すでに相対パス対応済み）

### 3. 動的コンテンツの扱い
- **Whisperモデル**:
  - 初回起動時にダウンロード
  - 保存場所: `%APPDATA%\whisper` または `~/.cache/whisper`
  - ダウンロード進行状況の表示確保（すでに実装済み）

- **設定ファイル**:
  - 保存場所: アプリケーションと同じディレクトリ
  - 現在の設定管理を維持（ConfigManager）
  - 破損時の回復メカニズムは既存のものを活用

### 4. PyInstaller設定
- **specファイル**: 既存のtranscriber.specを活用
- **hidden imports**: PyTorch、Whisper関連の動的インポートに注意
- **excludes**: 不要なモジュールの除外によるサイズ最適化
- **追加データファイル**:
  ```
  datas=[
    ('resources/*', 'resources'),
    ('ffmpeg_bin/*', 'ffmpeg_bin'),
  ]
  ```

### 5. 配布パッケージの構成
- **必要なファイル/フォルダ**:
  - 実行可能ファイル（コエモジ.exe）
  - resources/フォルダ
  - ffmpeg_bin/フォルダ
  - config.json（存在する場合）
  - README.txt（使用方法、初回起動時の注意事項など）

- **配布形式**:
  - 当面はZIPアーカイブ
  - 将来的にはInno SetupやNSISを使用したインストーラーを検討

### 6. 動作検証計画
- **テスト環境**: 
  - Windows 10（x64）
  - Windows 11（x64）
  - 異なるハードウェア仕様（低スペックマシンを含む）

- **検証シナリオ**:
  1. 初回起動テスト（モデルダウンロードを含む）
  2. 設定保存・読み込みテスト
  3. 音声ファイル処理テスト
  4. 動画ファイル処理テスト
  5. バッチ処理テスト
  6. エラーケースのテスト

## 実行計画

### ステップ1: PyInstallerのインストールと確認
```
pip install pyinstaller
pyinstaller --version
```

### ステップ2: specファイルの確認と調整
既存のtranscriber.specファイルを確認し、必要に応じて更新：
- リソースパスの設定
- 必要なモジュールの追加
- 不要なモジュールの除外

### ステップ3: PyInstallerを実行
```
pyinstaller transcriber.spec
```

### ステップ4: 生成物の確認
- dist/コエモジフォルダ内のファイル構成を確認
- 必要なリソースと依存ファイルがすべて含まれているか検証

### ステップ5: テスト実行
- 開発環境での動作確認
- 別環境（Pythonがインストールされていない環境）での動作確認

### ステップ6: 問題修正と再ビルド
- 発見された問題を修正
- 必要に応じて再ビルド

### ステップ7: 配布パッケージの作成
- 必要なファイルをまとめたZIPアーカイブの作成
- 簡易ユーザーマニュアル（README.txt）の作成

## 注意事項と考慮点

### サイズに関する考慮
- PyInstallerでビルドしたEXEは、Pythonインタープリタやライブラリを含むため大きなサイズになる
- PyTorchとWhisperを含めるとさらに大きくなる可能性がある
- 不要なモジュールを除外することでサイズを最適化

### パフォーマンスに関する考慮
- EXE化後も同等のパフォーマンスを維持できるか検証
- 特にGPU利用の機能が正常に動作するか確認

### バージョン管理
- EXE化したバージョンには明確なバージョン番号を付与
- 将来の更新に備えて配布物にもバージョン情報を含める
