# コエモジ - アクティブコンテキスト

## 現在の作業焦点

コエモジの開発は、基本機能の実装が完了し、「実行可能ファイル（EXE）化」と「配布準備」のフェーズに移行しています。特に以下の点を重点的に進めています：

1. **EXE化とパッケージング**: アプリケーションをPyInstallerを使用して実行可能ファイル化し、Python環境がないエンドユーザーでも使用できるようにする作業を進めています。

2. **FFmpegの同梱と自動検出**: FFmpegをアプリケーション内に同梱し、セットアップの簡略化とポータビリティの向上を実現。`ffmpeg_bin`フォルダ内のFFmpegを優先的に使用し、EXE化後も適切に動作するよう対応しています。

3. **アイコン設定の最適化**: アプリケーションのアイコン設定を改善し、EXE化されたアプリケーションでも高解像度のアイコンが表示されるよう対応。以下の優先順位でアイコンを選択：
   - 最優先: `resources/koemoji-logo256x256.ico`（高解像度アイコン）
   - 次点: `resources/koemoji-logo48x48.ico`（低解像度アイコン）
   - 最終手段: 動的に生成したアイコン

4. **Windows環境での安定性向上**: Windows専用アプリケーションとしての最適化を進め、特にEXE化後の動作の安定性確保に注力しています。

## 現在の意思決定と検討事項

1. **EXE化の具体的設計**
   - **パッケージング戦略**: ディレクトリ形式を採用（単一ファイル形式より起動が速く、FFmpegなどの外部バイナリを効率的に扱えるため）
   - **リソース管理**: 
     - アプリケーションアイコンとして`koemoji-logo256x256.ico`を使用
     - `resources/`フォルダのすべての画像ファイルを含める
     - `ffmpeg_bin/`フォルダをそのまま含める
   - **動的コンテンツの扱い**:
     - Whisperモデルは初回起動時にダウンロード（`%APPDATA%\whisper`または`~/.cache/whisper`）
     - 設定ファイルはアプリケーションと同じディレクトリに保存
   - **配布パッケージの構成**:
     - 実行可能ファイル（コエモジ.exe）
     - resources/フォルダ
     - ffmpeg_bin/フォルダ
     - README.txtなどの説明ファイル

2. **以前からの継続検討事項**
   - **クロスプラットフォーム対応の範囲**: 現状はWindows優先
   - **付加機能の優先順位**: 既存機能の安定化とEXE化を優先
   - **配布方法**: 当面はシンプルな配布パッケージ、将来的にはインストーラー作成も検討

## 主要なパターンと設計方針

開発を進める上で特に重視している設計方針は以下の通りです：

1. **モジュール分割と責任範囲の明確化**
   - UI、処理ロジック、設定管理の明確な分離
   - 各モジュールの責任領域の明確な定義

3. **ユーザー体験の一貫性**
   - エラー処理、進捗表示、フィードバックの一貫したパターン
   - 初回起動時のモデルダウンロードに対する適切なフィードバック

4. **配布と実行環境を考慮した設計**
   - PyInstaller用のファイルパス解決の対応
   - FFmpegなど外部バイナリの効率的な同梱と参照
   - リソースファイルの適切な取り込みと参照
