# コエモジ - アクティブコンテキスト

## 現在の作業焦点

コエモジの開発は、基本機能の安定化とセットアップの簡略化に集中しています。特に以下の点を重点的に改善中です：

1. **FFmpegの同梱と自動検出**：FFmpegをアプリケーション内に同梱し、セットアップの簡略化とポータビリティの向上を実現。最新の改善では、`ffmpeg_bin`フォルダ内のFFmpegを優先的に使用するよう実装しました。

2. **アイコン設定の最適化**：アプリケーションのアイコン設定を改善し、より高解像度のアイコンを優先的に使用するよう変更。以下の優先順位でアイコンを選択するように修正：
   - 最優先: `resources/koemoji-logo256x256.ico`（高解像度アイコン）
   - 次点: `resources/koemoji-logo48x48.ico`（低解像度アイコン）
   - 最終手段: 動的に生成したアイコン

3. **Windows環境での安定性向上**：Windows専用アプリケーションとしての最適化を進め、実行環境依存の問題を解消。

4. **現状機能の改善**：既存機能の品質向上と安定性の確保を優先し、新機能よりも既存機能の完成度を高めることに注力。

## 主要なパターンと設計方針

開発を進める上で特に重視している設計方針は以下の通りです：

1. **モジュール分割と責任範囲の明確化**
   - UI、処理ロジック、設定管理の明確な分離
   - 各モジュールの責任領域の明確な定義

2. **拡張可能性を重視した設計**

3. **ユーザー体験の一貫性**

   - 直感的に理解できるUI設計

4. **テスト可能な構造**
   - ビジネスロジックとUIの分離によるテスト容易性
   - モジュール単位でのテスト実施
